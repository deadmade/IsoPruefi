    Methods

    Login(Login)

    Authenticates a user and returns a JWT token for API access.

    public Task<ActionResult> Login(Login input)

    Parameters

    input Login
    The login credentials containing username and password.

    Returns

    JWT tokens and expiration information on successful authentication.

    Remarks:

    This endpoint validates user credentials and returns a JWT access token and refresh token. The returned tokens should be used for authenticating subsequent API requests.

    Example request: 

    {
"userName": "admin",
"password": "your-password"
}

    Example response: 

    {
"accessToken": "CakeIsNotALie.",
"refreshToken": "refresh-token-here",
"expiresIn": 3600
}

    Refresh(JwtToken)

    Refreshes an expired JWT access token using a valid refresh token.

    public Task<IActionResult> Refresh(JwtToken token)

    Parameters

    token JwtToken
    The JWT token object containing both the expired access token and valid refresh token.

    Returns

    A new set of JWT tokens if the refresh token is valid.

    Remarks:

    This endpoint allows clients to obtain a new access token without requiring the user to log in again. The refresh token must be valid and not expired.

    Use this endpoint when your access token expires to maintain continuous authentication without user intervention.

    Example request: 

    {
"accessToken": "expired-access-token",
"refreshToken": "valid-refresh-token"
}

    Example response: 

    {
"accessToken": "new-jwt-access-token",
"refreshToken": "new-refresh-token",
"expiresIn": 3600
}

    Register(Register)

    Registers a new user in the system. Admin access required.

    public Task<ActionResult> Register(Register input)

    Parameters

    input Register
    The registration data containing username and password for the new user.

    Returns

    Success confirmation when user is created successfully.

    Remarks:

    This endpoint allows administrators to create new user accounts in the system. Only users with the "Admin" role can access this endpoint.

    Authorization Required: Bearer token with Admin role

    Example request: 

    {
"userName": "newuser",
"password": "secure-password"
}

    The new user will be created with the "User" role by default and can access temperature data endpoints but cannot perform administrative functions.
